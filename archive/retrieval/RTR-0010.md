# RTR-0010
task_id: RTR-0010
title: Replace counter-based document IDs with stable canonical IDs
domain: retrieval
problem_type: violation
problem_statement: Chunk document IDs are derived from a global counter instead of the contract's deterministic identity inputs.
evidence_sources: [actual_truth/contracts/retrieval/index_structure_contract.md:133-155, tools/index/build_vector_index.py:235-280]
impacted_files: [tools/index/build_vector_index.py]
current_truth: `doc_id` values depend on traversal order and a process counter.
emergent_truth: Repeated builds can change document identity without content changes.
target_state: `doc_id` is stable for identical corpus content and derived from canonical identity inputs only.
verification_method: Rebuilding the same corpus yields the same ordered `doc_id` list, and a single changed chunk changes only its own `doc_id`.
dependencies: [RTR-0009]
launch_impact: critical
