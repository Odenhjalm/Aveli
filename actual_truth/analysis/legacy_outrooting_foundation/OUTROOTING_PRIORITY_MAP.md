# Outrooting Priority Map

This map orders the registered non-blocking drift by outrooting priority while preserving the locked canonical baseline boundary.

## Priority Rules

1. Do not target locked canonical baseline authority.
2. Start with residuals that most directly preserve alternate authority doctrine.
3. Defer lower-risk compatibility and support-surface residue until the boundary is already protected.

## HIGH

### DRIFT-002: Remote-schema and legacy migration lineage still encode non-canonical doctrine

- Highest concentration of legacy authority concepts.
- High risk of future confusion between canonical local baseline authority and non-authoritative migration residue.
- Recommended first outrooting branch: migration and historical doctrine cleanup planning only, without rewriting locked baseline slots.

### DRIFT-004: Home-audio legacy direct playback shaping still exists beside the canonical runtime path

- Adjacent alternate playback-shaping path near governed media runtime behavior.
- High remount and regression risk even while currently outside owned mounted scope.
- Recommended branch goal: prove non-mount status, then collapse into the contract-owned path.

### DRIFT-005: Legacy teacher_profile_media cleanup and migration residue still survives outside canonical profile-media authority

- Retired entity naming remains visible in cleanup and migration surfaces.
- High risk of accidental reuse during later profile/community work.
- Recommended branch goal: remove retired entity residue without touching the canonical `app.profile_media_placements` authority path.

## MEDIUM

### DRIFT-001: Stripe-era membership compatibility remains outside canonical app-entry authority

- No longer canonical, but still close to app-entry ownership.
- Recommended branch goal: collapse the compatibility layer after verifying no canonical runtime consumer depends on Stripe-shaped authority assumptions.

### DRIFT-003: Raw-table lesson and media helpers remain outside final mounted read authority

- Explicitly outside final mounted authority, but still reachable as raw-table helpers.
- Recommended branch goal: isolate or retire remaining sidecar callers after confirming no governed mounted surface depends on them.

### DRIFT-006: Legacy media payload fields still survive in upload, URL-normalization, and legacy verification surfaces

- Mostly support-surface and transition-payload residue.
- Recommended branch goal: replace legacy payload expectations once upstream outrooting removes the old authority models they preserve.

## Suggested Sequence

1. Protect the boundary: use `CANONICAL_VS_NONAUTHORITATIVE_BOUNDARY.md` before any outrooting tree is opened.
2. Attack doctrine-heavy residue first: `DRIFT-002`, `DRIFT-005`, `DRIFT-004`.
3. Collapse adjacent compatibility layers second: `DRIFT-001`, `DRIFT-003`.
4. Clean transitional payload and verification residue last: `DRIFT-006`.

## Non-Goals For This Map

- This map is not an implementation plan.
- This map does not authorize edits to locked baseline slots or active canonical contracts.
- This map does not replay baseline or perform a repo-wide diff.
