## 3. Test Data Identification Rule

Rows are eligible for test-data remediation only when all required predicates
match. This prevents false positives on real domain data.

Required predicates:

1. Target table/field:
   - `app.courses.title`

2. English test title pattern:
   - `title = 'Course ' || slug`

3. Slug pattern is one of:
   - `^bundle-stripe-fail-one-[0-9a-f]{6}$`
   - `^bundle-stripe-fail-two-[0-9a-f]{6}$`
   - `^bundle-course-[0-9a-f]{6}$`
   - `^bundle-course-bad-[0-9a-f]{6}$`

4. Teacher subject evidence:
   - `app.courses.teacher_id` joins to `app.auth_subjects.user_id`
   - subject email matches `^teacher_[0-9a-f]{6}@example\.com$`
   - subject role is `teacher`

5. At least one of the following test-context predicates:
   - linked `app.bundle_order_courses` row exists and linked order metadata is
     `{"checkout_type": "bundle"}`
   - slug matches `^bundle-course-bad-[0-9a-f]{6}$` and the course is
     `visibility='draft'` and `sellable=false`

Non-matching rows must not be remediated by this plan.

## 4. Remediation Strategy

Chosen strategy:

1. Clean up existing test residue in local DB.
2. Change test fixtures to use Swedish-safe domain titles.
3. Strengthen teardown so tests do not leave DB-owned product/domain content
   behind.
4. Update verification policy to fail when matching residue exists.

Option evaluation:

| Option | Decision | Justification |
|---|---|---|
| Delete test rows | Selected for existing local residue | The rows are generated test artifacts, not real domain data, and several are only retained by test snapshot dependencies. |
| Replace with Swedish-safe fixture values | Selected for future test-generated rows | Tests may still need course title values; generated values must be Swedish-safe at write time. |
| Isolate via environment or tagging | Selected as prevention layer | Test data should be tagged or isolated so cleanup can be deterministic and verification can distinguish fixture residue. |
| Exclude from verification via contract | Rejected as primary remediation | Exclusion would allow product-visible DB content to remain English and weakens TXT-004 language enforcement. |

Cleanup design:

- Scope: local/dev DB only unless a later task explicitly authorizes another
  environment.
- Operation type: controlled DB remediation task, not migration and not
  catalog rewrite.
- Selection: use the full identification rule above.
- Dependency traversal must preserve referential integrity.
- Cleanup must include, when present and only for selected rows:
  - `app.bundle_order_courses`
  - `app.orders`
  - `app.payments`
  - `app.payment_events`
  - `app.billing_logs`
  - `app.course_bundle_courses`
  - `app.course_bundles` only when owned exclusively by selected test subjects
    and no non-test courses remain attached
  - `app.course_enrollments`
  - `app.course_public_content`
  - `app.lessons`
  - `app.lesson_contents`
  - `app.courses`
  - generated `app.auth_subjects` and `auth.users` test subjects only when no
    non-remediation rows reference them
- Cleanup task must capture before/after counts and stable row hashes.
- Cleanup task must stop if any selected row is linked to non-test subject,
  non-test email, non-test order metadata, or non-matching slug/title pattern.

Replacement design for future tests:

- Replace `title = f"Course {slug}"` in course-bundle test helpers with a
  Swedish-safe deterministic fixture title.
- Required format:
  - `Paketkurs {suffix}` for normal bundle-course fixtures.
  - `Stripefel kurs {suffix}` for stripe-failure fixtures.
  - `Ej säljbar paketkurs {suffix}` for bad-course fixtures.
- Suffix may retain the existing hex suffix because it is identifier-like
  fixture disambiguation, not English product wording.
- Test assertions must not depend on English course title text.

## 5. Prevention Strategy

Test isolation model:

- Preferred: test-owned local DB residue must be removed after each test.
- Secondary: generated test rows must be deterministically classifiable.
- Verification must fail if classifiable test rows remain in DB-owned
  product/domain fields after test execution.

Teardown guarantees:

- Course-bundle tests must clean dependent rows before deleting courses.
- Cleanup must remove bundle order snapshots before course deletion when the
  snapshot is test-owned.
- Cleanup must delete orders/payment artifacts before deleting auth subjects.
- Cleanup must be idempotent.
- Cleanup must be safe when a test fails mid-flow.

Guardrails in test helpers:

- `_create_course` in `backend/tests/test_course_bundles.py` must not emit
  English product/domain titles.
- Test-generated slugs should keep deterministic prefixes for cleanup matching.
- Test-created users must keep deterministic `teacher_[hex]@example.com` and
  `student_[hex]@example.com` patterns or a stronger explicit fixture tag.
- Any future helper that writes `app.courses.title` must use Swedish-safe
  fixture titles.

Optional DB/policy guardrails:

- Do not add a product DB constraint forbidding English globally; real
  user-authored/domain content cannot be language-validated reliably with a
  simple constraint.
- A test-only guard may query for known English fixture patterns after backend
  test suites and fail if found.
- A local verification guard may require zero rows matching the identification
  rule before TXT-004 can pass Swedish DB-domain compliance.

## 6. Verification Policy Update

TXT-004-style verification must treat known test-generated DB-domain residue as
a failure unless an explicit local fixture audit task is in progress.

Policy:

- Do not exclude test residue from the Swedish-only DB-owned content gate by
  default.
- Require cleanup before claiming Swedish DB-domain compliance.
- Continue classifying `app.courses.title` as DB-owned `db_domain_content`.
- Continue checking English marker rows in `app.courses.title`.
- Add a deterministic test-residue subcheck using the identification rule in
  this artifact.

Verification outcomes:

| Condition | Outcome |
|---|---|
| No matching test residue and no English DB-domain values | `PASS_DB_OWNED_SAFE` |
| Matching test residue exists | `FAIL_TEST_DATA_LEAKAGE` |
| English DB-domain value exists but does not match test rule | `WARNING_LANGUAGE_MIXED` or `REAL_DOMAIN_DATA_REQUIRES_POLICY_DECISION` depending on evidence |
| Matching row links to non-test subject/order data | `ORIGIN_UNCERTAIN_BLOCKED` |
