# AOI-002.5 ADMIN BOOTSTRAP SERVICE_ROLE ACCESS SLOT

TYPE: `OWNER`  
TASK_TYPE: `BASELINE_SLOT`  
DEPENDS_ON: `["AOI-002"]`

## Goal

Create append-only baseline slot `0025_admin_bootstrap_service_role_access.sql` to align bootstrap function execution permissions with the operator/service_role access model.

## Required Outputs

- `REVOKE EXECUTE` on `app.bootstrap_first_admin(uuid)` from `public`, `anon`, and `authenticated`
- `GRANT EXECUTE` on `app.bootstrap_first_admin(uuid)` to `service_role`

## Forbidden

- modifying the function body of `app.bootstrap_first_admin(uuid)`
- introducing any new table, route, or execution authority
- exposing bootstrap through application runtime routes or frontend clients

## Exit Criteria

- `service_role` can execute the bootstrap function as an operator/tooling channel only
- `public`, `anon`, and `authenticated` cannot execute the bootstrap function
- authority ownership remains unchanged: backend is the only runtime authority
